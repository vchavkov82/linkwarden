# Impeccable Design Context — Linkwarden

## Design Context

### Users
Anyone from casual link collectors to researchers to small teams — a public, open-registration instance. Uses the product at a desk while browsing the web. Diverse technical backgrounds, ranging from enthusiasts who sought out self-hosting to first-time visitors discovering the tool. Core job: capture links fast, retrieve them reliably, organize without friction.

### Brand Personality
**Clean. Open. Inviting.**

Consumer-app warmth, not a power tool. The interface earns trust by staying out of the way and feeling pleasant to return to daily. Approachable before efficient — every element is there because it does something, but the overall feeling is welcoming rather than austere.

### Aesthetic Direction
- **Theme**: Light-mode primary, dark-mode fully polished as secondary
- **Reference**: No prescribed reference — forge something distinctive. Lean toward editorial/typographic minimalism rather than card-heavy SaaS aesthetics.
- **Not**: Purple gradients, glassmorphism, heavy drop-shadows, icon-above-every-heading grids, the Raindrop visual style
- **Color**: Sky-blue primary already established (`#0369a1` light / `#7dd3fc` dark). Keep it as anchor. Tint neutrals subtly toward it. Accents sparse.
- **Typography**: Functional hierarchy — one heading face with character, one body face that disappears into readability. Not Inter. Not Space Grotesk.
- **Density**: Medium. Enough breathing room to feel calm, enough density to feel productive. Not a spa, not a spreadsheet.

### Design Principles
1. **Content first** — links, collections, and tags are the product. Chrome exists only to serve them.
2. **Approachable utility** — tools feel like a good drawer organizer, not a command center. Easy to start from, powerful when you look closer.
3. **Consistent warmth** — slightly warm-tinted neutrals, generous whitespace, sky-blue as calm accent not loud highlight.
4. **Fast to scan** — typography and spacing create hierarchy; color is used sparingly for status and action, not decoration.
5. **Progressive disclosure** — simple surface, power available when you look closer. Advanced features feel like discoveries, not defaults.

### Tech Constraints
- Next.js 14 Pages Router, Tailwind CSS + DaisyUI, Radix UI primitives, Phosphor Icons
- Both light and dark DaisyUI themes — changes must respect both
- i18n via next-i18next — no hardcoded strings
